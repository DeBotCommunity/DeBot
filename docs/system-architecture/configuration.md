# Переменные окружения (.env)

Вся конфигурация DeBot осуществляется через переменные окружения, которые обычно хранятся в файле `.env` в корне проекта.

### Основные

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `API_ID` | Ваш API ID от Telegram. | - |
| `API_HASH` | Ваш API Hash от Telegram. | - |
| `USERBOT_ENCRYPTION_KEY` | 32-байтный ключ для шифрования данных в БД. **Генерируется автоматически при первой настройке.** | - |

### База данных

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `DB_TYPE` | Тип базы данных (диалект SQLAlchemy). | `postgresql` |
| `DB_HOST` | Адрес сервера базы данных. | `localhost` |
| `DB_PORT` | Порт сервера базы данных. | `5432` |
| `DB_NAME` | Имя базы данных. | `userbot_db` |
| `DB_USER` | Имя пользователя для подключения к БД. | `userbot` |
| `DB_PASS` | Пароль пользователя для подключения к БД. | `userbot_password` |
| `DB_CONN_RETRIES` | Количество попыток подключения к БД при старте. | `5` |
| `DB_CONN_RETRY_DELAY`| Задержка в секундах между попытками подключения. | `5` |

### Приложение

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `DEPLOY_TYPE` | Тип развертывания. `source` для сборки из исходников, `image` для использования готового образа. | `source` |
| `LOG_LEVEL` | Уровень логирования (DEBUG, INFO, WARNING, ERROR). | `INFO` |
| `GC_INTERVAL_SECONDS` | Интервал в секундах для принудительного вызова сборщика мусора. | `60` |

### Автообновление

Эти параметры имеют эффект только при `DEPLOY_TYPE=image`.

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `AUTO_UPDATE_ENABLED` | Включить автоматическое обновление (`True`/`False`). | `False` |
| `AUTO_UPDATE_INTERVAL_MINUTES` | Интервал в минутах для проверки и скачивания нового образа. | `1440` (24 часа) |
