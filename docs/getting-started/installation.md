# Установка и настройка

Прежде чем начать, убедитесь, что на вашем сервере или локальной машине установлены следующие компоненты:

-   **Git**
-   **Python 3.10** или выше
-   **Docker**
-   **Docker Compose**

## Шаг 1: Клонирование репозитория

Откройте терминал и выполните следующую команду, чтобы скачать исходный код проекта:

```bash
git clone https://github.com/DeBotCommunity/DeBot.git
cd DeBot
```

## Шаг 2: Интерактивная настройка

DeBot поставляется со скриптом, который поможет вам сгенерировать все необходимые конфигурационные файлы. Запустите его:

```bash
python3 -m scripts.setup
```

Скрипт задаст вам несколько вопросов:

1.  **Тип развертывания (`source` или `image`):**
    *   `source` (по умолчанию): Docker будет собирать образ из локальных исходных файлов. Выбирайте этот вариант, если вы планируете вносить изменения в код.
    *   `image`: Docker будет скачивать готовый, собранный образ из Docker Hub. Этот вариант быстрее и проще.
2.  **Настройка PostgreSQL:**
    *   Вы можете выбрать запуск базы данных PostgreSQL в отдельном Docker-контейнере (рекомендуется) или указать данные для подключения к уже существующей внешней БД.
3.  **Учетные данные Telegram:**
    *   Вам потребуется ввести ваш `API_ID` и `API_HASH`, которые можно получить на [my.telegram.org](https://my.telegram.org).

По завершении скрипт создаст два файла в корне проекта:

-   `.env`: Файл с вашими секретами и настройками (API ключи, пароли от БД, ключ шифрования).
-   `docker-compose.yml`: Готовый файл для запуска всего приложения.

## Шаг 3: Запуск контейнеров

Теперь, когда конфигурация готова, запустите проект с помощью Docker Compose:

```bash
docker-compose up -d
```

Эта команда скачает/соберет необходимые образы и запустит два контейнера в фоновом режиме: `debot_userbot` (сам бот) и `debot_postgres` (база данных, если вы выбрали этот вариант).

**Поздравляем!** DeBot запущен. Однако, он еще не управляет ни одним аккаунтом. Следующий шаг — добавить вашу первую сессию.
